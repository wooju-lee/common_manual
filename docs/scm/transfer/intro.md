# 📍 재고 로케이션 이동 (Transfer)

> 동일 매장 내에서 창고(Location) 간 재고를 이동하는 기능입니다.
>
> 경로 : `Inventory > Movement`

## AS-IS → TO-BE

| 구분 | AS-IS (Netsuite ERP) | TO-BE (IIC BO) |
|:---:|---|---|
| **재고 이동** | Netsuite에서 Inventory Transfer 직접 생성 | IIC BO에서 이동 등록 → Netsuite 자동 반영 |
| **이동 이력** | Netsuite 내 개별 조회 | IIC BO 이동 목록에서 통합 조회 |
| **재고 반영** | Netsuite 수동 반영 | 등록 즉시 출발·도착 창고 재고 자동 반영 |

:::tip 핵심 변경점
- **즉시 반영** — 이동 등록 시 출발 창고 차감 + 도착 창고 증가가 즉시 처리
- **자동 ERP 연동** — US/CA 매장의 경우 Netsuite에 Inventory Transfer 자동 생성
- **오류 자동 복구** — 후속 처리 실패 시 재고가 자동으로 원래 상태로 복구
:::

---

## 처리 절차

| Step | 처리 내용 |
|:---:|---|
| **1** | `Inventory > Movement` 메뉴 진입 후 **등록** 버튼 클릭 |
| **2** | 매장 선택 (본인 담당 매장만 선택 가능) |
| **3** | **출발 창고**와 **도착 창고** 선택 (반드시 서로 다른 창고) |
| **4** | 이동할 상품 코드와 수량(1개 이상) 입력 |
| **5** | 비고 입력 (선택 사항, 최대 500자) |
| **6** | **등록** 버튼 클릭 → 즉시 재고 반영 |

:::info 참고
재고 이동은 등록 즉시 반영됩니다. 별도의 확정(승인) 절차가 없으므로 수량 확인 후 등록해 주세요.
:::

---

### ERP 연동 (US/CA 매장)

US·CA 매장의 경우, 이동 등록 후 Netsuite에 Inventory Transfer가 **자동으로 생성**됩니다.

- 이 과정은 백그라운드로 처리되며, BO 화면에서는 즉시 반영됩니다.
- Netsuite 연동 실패 시 BO 재고는 자동 복구되며, 운영팀에 Slack 알림이 발송됩니다.

---

## 주요 제약 사항

| 항목 | 내용 |
|---|---|
| 같은 창고 불가 | 출발 창고와 도착 창고가 **동일하면 에러** |
| 재고 실사 중 차단 | 출발 또는 도착 창고 중 **하나라도 실사 중이면 이동 불가** |
| 상품 코드 중복 불가 | 동일 건 내 같은 상품 코드를 중복 입력할 수 없음 |
| 수량 제한 | 출발 창고의 보유 수량을 초과하여 이동할 수 없음 |
| 담당 매장만 가능 | 본인에게 권한이 부여된 매장만 이동 가능 |

## FAQ

### Q. Sales → DP 이동은 여기서 하나요?

> Sales(판매 재고) → DP(전시 재고) 이동은 **POS 일반 > 재고 이동** 메뉴에서 처리합니다.
> 해당 메뉴에서는 출발·도착 창고가 Sales(1000)→DP(1010)로 고정되어 있어 별도 선택이 필요 없습니다.
>
> 이 메뉴(`Inventory > Movement`)에서는 임의의 창고 간 이동이 가능합니다.

---

### Q. 이동 후 Netsuite에 바로 반영되나요?

> US·CA 매장의 경우, 이동 등록 후 Netsuite에 자동으로 반영됩니다. 다만 비동기 처리이므로 수 초 정도 지연이 있을 수 있습니다.
> Netsuite 연동에 실패하면 BO 재고가 자동 복구되며, Slack 알림이 발송됩니다.

---

### Q. 이동 취소는 가능한가요?

> 현재 이동 등록 후 취소 기능은 제공되지 않습니다.
> 잘못 이동한 경우, 반대 방향으로 다시 이동 등록하여 정정해야 합니다.
