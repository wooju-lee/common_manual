# 📍 재고 스냅샷 조회

> 특정 날짜 기준의 재고 현황을 조회하는 기능입니다. 매일 자동으로 생성됩니다.
>
> 경로 : `Inventory > Inventory Snapshot`

## AS-IS → TO-BE

| 구분 | AS-IS (Netsuite ERP) | TO-BE (IIC BO) |
|:---:|---|---|
| **특정 시점 재고 확인** | Netsuite Saved Search로 개별 조회 | IIC BO에서 날짜 선택으로 간편 조회 |
| **스냅샷 생성** | 수동 | 매일 자동 생성 (매장 시간대 기준 23시) |
| **SAP 연동** | 별도 수동 작업 | 자동으로 SAP에 스냅샷 데이터 전송 |
| **WMS 재고 대조** | 수동 대조 | WMS(Pantos) 스냅샷 자동 수신 및 저장 |

:::tip 핵심 변경점
- **자동 생성** — 매일 매장 시간대 기준 23시에 자동 스냅샷 생성
- **과거 재고 확인** — 날짜만 선택하면 해당일의 재고 상태를 바로 확인 가능
- **다중 소스** — Netsuite, WMS(Pantos) 재고를 통합하여 하나의 스냅샷으로 관리
:::

## 조회 방법

| Step | 처리 내용 |
|:---:|---|
| **1** | `Inventory > Inventory Snapshot` 메뉴 진입 |
| **2** | **매장 선택** (필수, 복수 선택 가능) |
| **3** | **조회 날짜 선택** (필수) |
| **4** | 창고, 상품 검색 등 추가 조건 설정 (선택) |
| **5** | 조회 실행 |
| **6** | 필요 시 엑셀 다운로드 |

### 검색 조건

| 조건 | 설명 | 필수 여부 |
|---|---|:---:|
| 브랜드 | 브랜드 선택 | 선택 |
| 법인 | 법인 선택 | 선택 |
| **매장** | 매장 코드 선택 (복수 가능) | **필수** |
| 창고 | 창고(Location) 선택 | 선택 |
| **조회 날짜** | 스냅샷 기준 날짜 | **필수** |
| 검색어 | 상품코드, 상품명, 바코드 (2자 이상) | 선택 |

:::caution 주의
매장과 날짜는 **필수 입력** 항목입니다. 미입력 시 조회가 실행되지 않습니다.
:::

## 스냅샷 자동 생성 시점

스냅샷은 다음 시점에 자동으로 생성됩니다.

| 소스 | 생성 시점 | 대상 |
|:---:|---|---|
| **IIC BO** | 매장 시간대 기준 매일 자동 | 전체 매장 |
| **Netsuite** | 매장 시간대 기준 23시 | CA + US 물류 매장 |
| **Pantos WMS** | WMS에서 전송 시 | US 물류 창고 |

:::info 참고
스냅샷은 해당 시점의 재고를 "사진 찍듯" 저장한 것이므로, 조회 시점의 실시간 재고와 다를 수 있습니다.
실시간 재고는 `Inventory > Realtime Inventory` 메뉴를 이용해 주세요.
:::

## FAQ

### Q. 스냅샷이 없는 날짜를 조회하면 어떻게 되나요?

> 해당 날짜에 스냅샷이 생성되지 않았다면 조회 결과가 비어 있습니다.
> 스냅샷은 매일 자동 생성되지만, 시스템 점검 등의 사유로 누락될 수 있습니다.

---

### Q. 실시간 재고 조회와 차이가 나는 이유는 무엇인가요?

> 스냅샷은 **특정 시점의 재고를 저장한 것**이므로, 해당 시점 이후의 입출고/이동/조정이 반영되지 않습니다.
> 최신 재고는 실시간 재고 조회 메뉴에서 확인해 주세요.

---

### Q. SAP로 전송되는 스냅샷은 어떤 건가요?

> CA 매장의 경우, 매장 시간대 기준 새벽 3시에 전일자 스냅샷이 SAP로 자동 전송됩니다.
> 별도의 수동 작업은 필요 없습니다.
