# 📍 재고 조정 (Adjustment)

> 재고 수량을 수동으로 증감 조정하는 기능입니다.
>
> 경로 : `Inventory > Adjustment`

## AS-IS → TO-BE

| 구분 | AS-IS (Netsuite ERP) | TO-BE (IIC BO) |
|:---:|---|---|
| **조정 등록** | Netsuite에서 Inventory Adjustment 직접 생성 | IIC BO에서 조정 등록 → 승인 후 반영 |
| **승인 프로세스** | 별도 승인 없이 즉시 반영 | 등록(대기) → 확정/반려 2단계 처리 |
| **재고 반영 시점** | 등록 즉시 | 확정 시점에 반영 (대기 중에는 가용 재고만 임시 차감) |
| **조정 이력** | Netsuite 내 조회 | IIC BO 조정 목록에서 통합 조회 |

:::tip 핵심 변경점
- **승인 프로세스 추가** — 등록 → 확정 2단계로 관리하여, 잘못된 조정이 바로 반영되는 것을 방지
- **가용 재고 관리** — 조정 대기 중에도 해당 수량이 가용 재고에서 임시 차감되어 이중 처리 방지
- **Netsuite 연동 불필요** — ERP 접속 없이 BO에서 전체 조정 프로세스 관리 가능
:::

---

## 조정 상태

```
대기(Ready) → 확정(Confirmed)
           → 반려(Rejected)
```

| 상태 | 설명 |
|:---:|---|
| **대기 (Ready)** | 조정 등록 완료. 가용 재고에서 임시 차감됨. 승인 대기 중 |
| **확정 (Confirmed)** | 승인 완료. 실제 보유 수량이 변경됨 |
| **반려 (Rejected)** | 조정 거부. 임시 차감이 해제되어 가용 재고 복구 |

:::info 참고
**대기** 상태에서는 실제 보유 수량(onHand)은 변하지 않고, **가용 재고**만 임시 차감됩니다.
확정 시 실제 보유 수량이 변경됩니다.
:::

## 처리 절차

### 조정 등록

| Step | 처리 내용 |
|:---:|---|
| **1** | `Inventory > Adjustment` 메뉴 진입 후 **등록** 버튼 클릭 |
| **2** | 매장과 창고 선택 |
| **3** | 상품별 조정 수량 입력 |
| **4** | 조정 사유 코드 선택 (6자리) |
| **5** | 비고 입력 (선택, 최대 200자) |
| **6** | 필요 시 청구 매장·창고 선택 (같은 법인 내만 가능) |
| **7** | **등록** 버튼 클릭 → 대기 상태로 생성 |

**조정 수량 입력 규칙**

| 사유 유형 | 입력 가능 방향 | 설명 |
|:---:|:---:|---|
| 증감 모두 가능 (BOTH) | +, - | 양수(증가) 또는 음수(감소) 모두 입력 가능 |
| 증가만 가능 | + | 양수만 입력 가능 |
| 감소만 가능 | - | 시스템에서 자동으로 음수 처리 |

### 엑셀 일괄 등록

| Step | 처리 내용 |
|:---:|---|
| **1** | 조정 화면에서 **엑셀 업로드** 버튼 클릭 |
| **2** | 엑셀 양식에 맞게 작성된 파일 업로드 (xlsx, 최대 5MB) |
| **3** | 결과 확인 (오류 건은 사유와 함께 표시) |

### 조정 확정 / 반려

| Step | 처리 내용 |
|:---:|---|
| **1** | 조정 목록에서 대기 상태인 건 선택 |
| **2-A** | **확정** → 실제 보유 수량에 반영. 재고 원장에 이력 기록 |
| **2-B** | **반려** → 가용 재고 복구. 보유 수량 변경 없음 |

:::caution 주의
일괄 확정 시, 선택한 건 중 **하나라도 대기 상태가 아닌 건이 포함되면 전체가 실패**합니다.
대기 상태인 건만 선택해 주세요.
:::

## 주요 제약 사항

| 항목 | 내용 |
|---|---|
| 사유 코드 필수 | 6자리 사유 코드를 반드시 선택해야 함 |
| 비고 길이 | 최대 200자 |
| 청구 매장 법인 | 청구 매장은 조정 매장과 **같은 법인**이어야 함 |
| 확정 권한 | 조정 등록과 확정(승인)은 **별도 권한**으로 분리 |

## FAQ

### Q. 조정을 등록하면 바로 재고에 반영되나요?

> 아닙니다. 조정 등록 시에는 **대기** 상태로 생성되며, 가용 재고만 임시 차감됩니다.
> 실제 보유 수량은 **확정** 처리 후에 변경됩니다.

---

### Q. 입고 확정 시 수량 차이로 자동 생성된 조정은 어떻게 되나요?

> 법인간(C2C) 입고의 상세 확정에서 예정 수량과 실제 수량이 차이 나는 경우, 시스템이 자동으로 조정 건을 생성합니다.
> 이 자동 생성 조정은 별도 확정 없이 **즉시 재고에 반영**됩니다.

---

### Q. 조정 등록 후 수정할 수 있나요?

> 대기 상태에서는 확정/반려 시 수량 등 조정 정보를 함께 수정할 수 있습니다.
> 이미 확정된 건은 수정할 수 없으며, 반대 방향의 조정을 새로 등록해야 합니다.
