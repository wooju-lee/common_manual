# 📍 재고 원장 조회

> 모든 재고 처리 이력에 대해 조회할 수 있는 메뉴입니다.
>
> 경로 : `Inventory > Inventory Ledger`

## AS-IS → TO-BE

| 구분 | AS-IS (ERP, Old IPOS) | TO-BE (IIC BO) |
|:---:|---|---|
| **전체 재고 처리 이력 조회** | 한번에 조회할 수 있는 영역 없음 | 재고 원장 조회 |
| **이력 유형 필터** | 시스템별 개별 확인 필요 | 판매/환불/입고/출고/조정/실사/이동 통합 필터 |
| **수량 변동 추적** | 개별 확인 필요 | 변경 전 → 변동 → 변경 후 한눈에 확인 |

:::tip 핵심 변경점
- **통합 원장** — 판매, 환불, 입고, 출고, 조정, 실사, 이동 등 모든 재고 변동을 하나의 화면에서 조회
- **수량 변동 추적** — 각 이력에 변경 전 수량 / 변동 수량 / 변경 후 수량이 표시되어 흐름 파악이 용이
- **오류 복구 이력** — 시스템 오류 시 자동 복구된 이력도 선택적으로 조회 가능
:::

---

## 조회 방법

| Step | 처리 내용 |
|:---:|---|
| **1** | `Inventory > Inventory Ledger` 메뉴 진입 |
| **2** | **시작일 / 종료일** 설정 (필수) |
| **3** | 법인, 매장, 창고 등 추가 조건 설정 (선택) |
| **4** | 재고 처리 유형 필터 선택 (선택) |
| **5** | 조회 실행 |
| **6** | 필요 시 엑셀 다운로드 |

### 검색 조건

| 조건 | 설명 | 필수 여부 |
|---|---|:---:|
| **시작일** | 조회 시작 날짜 | **필수** |
| **종료일** | 조회 종료 날짜 (시작일 이후) | **필수** |
| 법인 | 법인 코드 선택 (복수 가능) | 선택 |
| 매장 | 매장 코드 선택 (복수 가능) | 선택 |
| 창고 | 창고(Location) 선택 (복수 가능) | 선택 |
| 처리 유형 | 아래 유형 중 선택 (복수 가능) | 선택 |
| 복구 이력 포함 | 오류 복구(보상) 이력 표시 여부 | 선택 (기본: OFF) |
| 검색어 | Invoice No, 상품코드, 상품명, 바코드 (2자 이상) | 선택 |

## 재고 처리 유형

| 유형 | 설명 | 포함되는 이력 |
|:---:|---|---|
| **판매 (Sale)** | POS/Shopify 판매 | 일반 판매, RX 판매 |
| **환불 (Refund)** | 환불 처리 | 환불에 의한 재고 복구 |
| **SAP 입고** | SAP 기반 입고 | 법인 초기 재고 입고 |
| **수기 입고** | BO에서 직접 등록한 입고 | 스토어 수기 입고, 엑셀 업로드 입고 |
| **출고 (Outbound)** | 출고 처리 | 출고 요청 예약 + 실 출고 차감 |
| **조정 (Adjustment)** | 재고 조정 | 입고 차이 조정, 일반 조정, 실사 조정 |
| **실사 (Stocktaking)** | 재고 실사 | 실사에 의한 수량 변경 |
| **이동 (Movement)** | 창고 간 이동 | 일반 이동, RX 이동 |

:::info 복구 이력이란?
시스템 오류 등으로 재고 처리가 실패했을 때, 자동으로 원래 상태로 복구한 이력입니다.
기본적으로는 표시되지 않으며, **"복구 이력 포함"** 옵션을 ON으로 설정하면 함께 조회됩니다.
:::

## 조회 결과 항목

| 항목 | 설명 |
|---|---|
| 처리 일시 | 재고 변동이 발생한 날짜/시간 (매장 시간대 기준) |
| 법인 / 매장 / 창고 | 재고 변동이 발생한 위치 |
| 상품 정보 | 상품 코드, 상품명 |
| 처리 유형 | 판매, 환불, 입고, 출고, 조정, 실사, 이동 |
| **변경 전 수량** | 처리 전 보유 수량 |
| **변동 수량** | 증가(+) 또는 감소(-) 수량 |
| **변경 후 수량** | 처리 후 보유 수량 |

### 정렬 가능 항목

생성일, 법인 정보, 매장 정보, 상품 정보, 변경 전 수량, 변동 수량, 변경 후 수량

## FAQ

### Q. 원장에 표시되는 시간은 어느 시간대 기준인가요?

> **매장 시간대(타임존)** 기준으로 표시됩니다.
> 예) US 매장의 이력은 해당 매장의 로컬 시간으로 표시됩니다.

---

### Q. 판매와 환불이 모두 보고 싶으면 어떻게 하나요?

> 처리 유형 필터에서 **판매**와 **환불**을 복수 선택하면 됩니다.
> 필터를 선택하지 않으면 전체 유형이 조회됩니다.

---

### Q. 재고가 맞지 않을 때 원인을 어떻게 찾나요?

> 1. 해당 매장·창고·상품 코드로 기간 설정 후 원장 조회
> 2. 변경 전/후 수량의 흐름을 추적하여 어느 시점에서 차이가 발생했는지 확인
> 3. 처리 유형 필터를 활용하여 특정 유형(출고, 조정 등)만 필터링하여 상세 확인
> 4. 필요 시 "복구 이력 포함"을 ON으로 설정하여 시스템 오류 복구 건도 함께 확인
