# 📍 출고 (Transfer Order)

> 매장 간 또는 매장 ↔ 물류 간 상품을 출고하는 기능입니다.
>
> 경로 : `Inventory > Outbound`

## AS-IS → TO-BE

| 구분 | AS-IS (Netsuite ERP) | TO-BE (IIC BO) |
|:---:|---|---|
| **출고 등록** | Netsuite에서 Transfer Order 직접 생성 | IIC BO에서 출고 등록 → Netsuite·WMS 자동 연동 |
| **출고 요청/승인** | 별도 승인 프로세스 없음 | 출고 요청 → 승인자 확정 후 처리 (선택 가능) |
| **WMS 연동** | 수동 또는 별도 시스템 | 출고 등록 시 WMS 자동 출고 요청 |
| **출고 상태 추적** | Netsuite 내 수동 확인 | BO에서 실시간 상태 추적 (예정 → 등록 → 완료) |
| **재고 반영** | Netsuite 수동 반영 | 출고 완료 시 BO·ERP 재고 자동 차감 + 도착지 입고 자동 생성 |

:::tip 핵심 변경점
- **수동 → 자동** — 출고 등록만 하면 Netsuite TO 생성, WMS 출고 요청, 상태 업데이트가 모두 자동 처리
- **승인 프로세스 추가** — 출고 요청 → 승인 워크플로우를 통해 관리 강화 (선택 사항)
- **통합 이력** — 출고·입고가 연결되어 상품 이동 전체를 한눈에 확인 가능
:::

## 출고 유형

| 유형 | 설명 | 출발지 → 도착지 |
|:---:|---|---|
| **C2C** | 법인 간 이동 | 법인 본사 → 법인 본사 |
| **S2S** | 매장 간 이동 | 매장 → 매장 (같은 법인 내) |
| **L2S** | 물류 → 매장 | 물류 창고 → 매장 |
| **S2L** | 매장 → 물류 | 매장 → 물류 창고 |

:::caution 주의
S2S, L2S, S2L은 **같은 법인 내 매장** 사이에서만 출고가 가능합니다. C2C는 법인 간 이동이므로 제한 없습니다.
:::

## 출고 상태

```
예정(Planned) → 대기(Pending) → 등록완료(Registered) → 완료(Complete)
                                                        ↘ 취소(Canceled)
```

| 상태 | 설명 |
|:---:|---|
| **예정 (Planned)** | 출고 등록 완료. ERP/WMS 연동 전 대기 상태 |
| **대기 (Pending)** | ERP/WMS에 출고 요청 전송됨 |
| **등록완료 (Registered)** | WMS에서 출고 접수 확인 (Accepted) |
| **완료 (Complete)** | WMS 출고 완료. 재고 차감 및 도착지 입고 자동 생성 |
| **취소 (Canceled)** | 출고 취소됨 |

:::info 참고
ERP/WMS 연동이 없는 국가(BO-Only)에서는 출고 등록 시 바로 **완료** 처리되며, 도착지 입고가 즉시 생성됩니다.
:::

## 처리 절차

### 방법 1. 출고 등록 (바로 처리)

| Step | 처리 내용 |
|:---:|---|
| **1** | `Inventory > Outbound` 메뉴 진입 후 **등록** 버튼 클릭 |
| **2** | 출고 유형(C2C/S2S/L2S/S2L) 선택 |
| **3** | 출발지 매장·창고, 도착지 매장·창고 선택 |
| **4** | 출고 예정일 선택 |
| **5** | 출고할 상품 코드와 수량 입력 (최대 50개 상품) |
| **6** | **등록** 버튼 클릭 → ERP/WMS 자동 연동 시작 |

### 방법 2. 출고 요청 (승인 후 처리)

| Step | 처리 내용 |
|:---:|---|
| **1** | 위 1~5 과정 동일 |
| **2** | **출고 요청**으로 등록 → 해당 수량이 예약(가용 재고에서 차감) |
| **3** | 승인자가 `Inventory > Outbound Request` 에서 요청 건 확인 |
| **4** | **승인** 시 정규 출고 건이 생성되어 연동 시작 / **반려** 시 예약 해제 |

### 방법 3. 엑셀 일괄 등록

| Step | 처리 내용 |
|:---:|---|
| **1** | 출고 화면에서 **엑셀 업로드** 버튼 클릭 |
| **2** | 엑셀 양식에 맞게 작성된 파일 업로드 |
| **3** | 결과 확인 (오류 건은 사유와 함께 표시) |

---

### WMS 출고 완료 후 처리

WMS(Pantos)로부터 상태 알림을 수신하면 BO에 자동으로 반영됩니다.

| WMS 상태 | BO 처리 |
|:---:|---|
| `Accepted` | 출고 상태 → **등록완료** |
| `Completed` | 출고 상태 → **완료**. 출발지 재고 차감 + 도착지 입고 자동 생성 |
| `Canceled` | 출고 상태 → **취소**. Netsuite Transfer Order 닫힘 처리 |

## 주요 제약 사항

| 항목 | 내용 |
|---|---|
| 상품 수 제한 | 출고 1건당 **최대 50개 SKU** |
| 같은 법인 필수 | S2S, L2S, S2L은 **같은 법인** 내에서만 가능 |
| 재고 실사 중 차단 | 출발지 창고에서 재고 실사 진행 중이면 **출고 불가** |
| 상품 코드 중복 불가 | 동일 출고 건 내 같은 상품 코드 중복 입력 불가 |

## FAQ

### Q. 출고 등록과 출고 요청의 차이는 무엇인가요?

> **출고 등록**은 바로 출고가 진행되며, **출고 요청**은 승인자의 확인 후에 출고가 진행됩니다.
> 출고 요청 시 해당 수량은 가용 재고에서 차감(예약)되므로, 다른 출고와 중복되지 않습니다.

---

### Q. 출고 완료 후 도착지에 자동으로 입고가 생기나요?

> 네. 출고 완료 시 도착지에 대응 입고(Inbound)가 자동 생성됩니다.
> 해당 입고 건은 `Inventory > Inbound` 메뉴에서 확인 및 확정 처리할 수 있습니다.

---

### Q. 재고 실사 중인 창고에서 출고하려면 어떻게 하나요?

> 재고 실사가 완료된 후에 출고가 가능합니다. 실사 중에는 재고 정합성 보장을 위해 해당 창고의 출고/이동이 일시 차단됩니다.
