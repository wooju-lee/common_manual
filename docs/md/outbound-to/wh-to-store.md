# W.H To Store (L2S)

> 입고 받은 창고 재고를 각 스토어에 분배하는 출고 오더를 생성하는 출고 타입입니다.
>
> 경로 : `Inventory > Outbound`

---

![W.H To Store 화면](./스크린샷%202026-04-16%2017.53.18.png)

---

## 출고 등록

`Inventory > Outbound` 에서 **W.H → Store** 유형의 출고를 생성합니다.

:::tip 실시간 재고 확인
출고 정보를 생성하기 전, **실시간 재고 정보를 먼저 확인**하는 것을 권장합니다.
:::

### 출고 수량과 재고 반영

| 시점 | 재고 처리 |
|:---:|---|
| 출고 생성 ~ 출고 완료 전 | 해당 수량이 **출고 대기 수량**으로 커밋 (재고 미차감) |
| **출고 완료** 시 | 창고 재고에서 **실제 차감** |

---

## 출고 완료 처리

W.H(창고)에서 실제 출고가 완료되면 WMS로부터 `Completed` 상태값이 전송되어 **출고 완료** 상태로 자동 업데이트됩니다.

:::caution 재고 부족 시
창고 재고가 부족하여 출고가 불가능한 경우, 등록된 출고를 **취소**한 뒤 실제 가용 재고에 맞춰 **신규 출고 오더를 재생성**할 수 있습니다. *(TBD)*
:::

---

## 입고 자동 생성

출고 완료 시, 도착지 스토어 기준으로 **입고 리스트가 자동 생성**됩니다.

| Step | 처리 내용 |
|:---:|---|
| **1** | 출고 완료 → 입고 스토어 기준 **입고 리스트 생성** |
| **2** | 스토어에서 **입고 확정** 처리 |
| **3** | 출고 수량이 해당 스토어의 **입고 재고로 반영** |

---

## 전체 흐름 요약

```
출고 등록 → 출고 대기 (수량 커밋) → W.H 실출고 → 출고 완료 (재고 차감)
                                                    ↓
                                         입고 리스트 자동 생성 → 입고 확정 → 스토어 재고 반영
```
